# Dev Note 파일 로드

args = `{{ args }}`

args가 비어 있으면 **Branch A(목록 표시)**, 파일명이 있으면 **Branch B(직접 로드)**로 분기합니다.

---

## Branch A — args 없음: 파일 목록 표시

### 1단계 — 파일 목록 및 완료 여부 수집

1. Glob으로 `.claude/.dev_note/*.md` 파일 목록을 가져옵니다
2. `.claude/.dev_note/.completed` 파일을 Read합니다
   - 각 줄이 완료된 devnote 파일명입니다 (예: `1.0.11_ver.md`)
   - 파일이 없거나 읽기 실패 시: 완료 여부 판단 없이 파일 목록만 표시합니다

### 2단계 — 목록 표시

완료 여부 기준으로 그룹화하여 출력합니다. 각 그룹 내부는 버전 번호 내림차순(최신 먼저)으로 정렬합니다:

```
.claude/.dev_note/ 파일 목록

[ 미완료 ]
 1. 1.0.12_ver.md

[ 완료 ]
 2. 1.0.11_ver.md
 3. 1.0.11_bugfix_3.md
 ...

번호 또는 파일명을 입력하면 해당 devnote를 불러옵니다.
```

### 3단계 — 사용자 선택 대기

목록을 표시한 후 **사용자의 입력을 기다립니다.**

- 번호 입력 → 해당 번호 파일을 Branch B 로직으로 로드합니다
- 파일명 입력 → 해당 파일명을 Branch B 로직으로 로드합니다

---

## Branch B — args 있음 (또는 Branch A에서 선택): 파일 로드

### 1단계 — 파일 경로 결정 및 읽기

1. args(또는 Branch A 선택)에서 파일명을 추출합니다
2. `.md`로 끝나지 않으면 `.md`를 붙입니다
3. `.claude/.dev_note/<파일명>.md` 경로로 Read 툴을 호출합니다
4. 파일이 없으면 오류를 보고하고 Branch A의 목록 표시로 전환합니다

### 2단계 — 파일 내용 분석

파일을 읽은 후 다음을 파악합니다:

- **작업 유형**: 기능 개선 / 버그 수정 / 리팩토링 / 문서화 등
- **사용할 스킬**: 파일 내 `사용할 skills:` 또는 `skills:` 키워드로 명시된 스킬 이름
  - 명시된 스킬이 없으면 작업 유형에 맞게 추천 (`feature-dev`, `micro-fix` 등)
- **작업 목록**: 구현해야 할 항목들을 번호 목록으로 정리
- **주요 파일**: 영향 받을 것으로 예상되는 파일/모듈

### 3단계 — 사용자에게 브리핑

다음 형식으로 사용자에게 요약을 제공합니다:

```
파일: .claude/.dev_note/<파일명>.md

작업 유형: <유형>
사용할 스킬: <스킬명>

작업 항목:
1. ...
2. ...

"진행시켜" 또는 원하는 작업 번호를 말씀해주세요.
```

### 4단계 — 사용자 지시 대기

브리핑 후 **반드시 사용자의 지시를 기다립니다.** 자동으로 작업을 시작하지 않습니다.

- 사용자가 "진행시켜" / "go" / "시작해" 등 전체 진행을 요청하면 → 명시된 스킬을 Skill 툴로 호출합니다
  - **args 구성 (중요)**: `devnote:<파일명.md>` 을 첫 줄에 반드시 포함하고, 그 아래에 작업 내용 요약을 전달합니다
    ```
    devnote:1.0.12_ver.md
    <작업 내용 요약>
    ```
- 사용자가 특정 번호나 항목을 지정하면 → 해당 항목만 진행합니다 (동일한 args 구성 사용)
- 사용자가 추가 질문을 하면 → 파일 내용을 바탕으로 답변합니다

### 주의

- 부하 테스트 / 성능 측정 항목은 `load-test-policy.md` 규칙에 따라 자동 실행하지 않고 사용자에게 고지합니다
