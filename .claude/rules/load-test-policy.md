# 부하 / 성능 테스트 자동 실행 금지

부하 테스트·성능 테스트·대량 데이터 측정은 **사용자가 명시 요청할 때만** 수행합니다. 패치 노트·리뷰·체크리스트에 등장해도 자동 실행하지 않습니다.

## 자동 실행 금지 트리거

- 부하 테스트, stress test, 벤치마크, loadtest, throughput/latency/rps/tps 측정
- 수백 건 이상 단위 import/export **실측 시간 측정** (기능 동작 검증은 OK)
- 동시 N명 시뮬레이션, 이벤트 루프 lag 측정
- `autocannon`, `k6`, `wrk`, `artillery` 등 부하 도구
- `--prof`, `--cpu-prof` 등 V8 프로파일러

## 허용 범위 (자동 실행 OK)

단위 테스트, 소규모 통합 테스트(~10건 fixture), 타입 체크, 린트, 빌드, dry-run 검증.  
**"맞게 동작하는가" = OK, "얼마나 빨리/많이 처리하는가" = 자동 금지.**

## 발견 시 행동

1. 해당 단계 **건너뛰고** 진행
2. 사용자에게 한 줄 알림: "부하 측정 항목은 토큰 비용 정책상 자동 실행하지 않았습니다."
3. 결과 보고에서 `skipped (load-test policy)` 표기 — 수행한 것처럼 추정·창작 금지
